# PR-First Delivery Workflow

1. State the objective.
2. Generate an implementation plan.
3. Check out to a new branch (e.g., `git checkout -b feature/task-name`).
4. Execute [MODIFY] and [NEW] file operations.
5. Commit changes with a descriptive message (`git add . && git commit -m "feat: descriptive message"`).
6. Create a Pull Request for the branch instead of pushing to main.
7. Present the PR and diffs for final architectural inspection.
