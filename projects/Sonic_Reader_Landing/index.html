FAQ

Questions?

Does it work with any application?

Sonic Reader uses Windows UI Automation to read text from any window on your screen — browsers, PDFs, Word docs, emails, IDEs, even terminal windows. If you can see it, Sonic can read it.

Is it really 100% offline?

Yes. The voice engine (Piper TTS) runs entirely on your machine. No internet connection required. No data is sent anywhere. Your text stays on your computer.

What about the Chrome Legacy Window bug?

V2 includes a native Rust fix that recursively searches for the real browser content element, bypassing the legacy accessibility stub completely. This was the #1 crash cause in V1 — it's gone.

Can I use it while working?

Absolutely. Sonic Reader floats as an always-on-top compact window. It stays out of your way and never interferes with your workflow. The Ghost Overlay ensures it's invisible to its own scraper.

What makes V2 different from V1?

Complete rewrite. New Rust backend with process lifecycle management (zero zombie processes), a 5-stage text sanitizer that handles code and symbols, and a premium React UI built with Tauri 2.0.

Windows only?

Currently yes, because the scraper relies on Windows UI Automation. macOS support via the Accessibility API is on the roadmap.
